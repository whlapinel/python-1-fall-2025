Assignment 1.2: Syntax Error Detective

Due: Thursday, August 15 (before class) Type: Prepare Points: 20

Overview

Practice identifying and fixing syntax errors while understanding the difference between syntax and semantic issues in Python code. This assignment reinforces PCEP 1.1 objectives on Python’s lexis, syntax, and semantics.

Part 1: Syntax Error Hunt (10 points)

Each code sample below contains exactly one syntax error. Your job is to:

  1. Identify the error
  2. Explain what’s wrong
  3. Provide the corrected code
  4. Test your fix in Python to make sure it works

Sample Format:

Original Code: print "Hello World"
Error: Missing parentheses around the argument
Corrected Code: print("Hello World")
Tested: ✓ (checkmark when you've tested it)

  • Syntax Error: Python can’t understand the code (won’t run)

  2. Test all your fixes in Python before submitting

Grading Rubric

  • Part 1 (10 pts): 1 point per correctly identified and fixed syntax error
  • Part 2 (5 pts): 1 point per correct classification of error type
  • Part 3 (5 pts): 1 point per accurate error message and explanation

Tips for Success

  • Use Python to test everything - don’t just guess!
  • Read error messages carefully - they often tell you exactly what’s wrong
  • Remember the difference:
    • Syntax = grammar rules (will Python understand this?)
    • Semantics = meaning (does this do what I intended?)
  • Pay attention to: quotes, parentheses, colons, indentation, variable names
